## 1. Goals

Build a Docusaurus-powered personal brand site with four first-class content sections:

| Section  | Meaning                                                                     | Content source    |
| -------- | --------------------------------------------------------------------------- | ----------------- |
| Research | Long-form studies / analyses / papers                                       | First-party       |
| Featured | Appearances on other people's platforms (interviews, podcasts, guest posts) | Third-party links |
| Insights | Posts and ideas — blog-style                                                | First-party       |
| Newsroom | Announcements and milestones                                                | First-party       |

> **Naming:** "Featured" (per user decision) for third-party appearances on other people's platforms. Route: `/featured`.

## 3. Docusaurus Mapping

Docusaurus supports **multiple instances** of the same plugin. The site uses three blog instances (no docs plugin at all) plus a custom showcase page:

| Section  | Plugin                                         | Instance ID             | Route       |
| -------- | ---------------------------------------------- | ----------------------- | ----------- |
| Research | `@docusaurus/plugin-content-blog`              | `research`              | `/research` |
| Insights | `@docusaurus/plugin-content-blog`              | `default` (preset slot) | `/insights` |
| Newsroom | `@docusaurus/plugin-content-blog`              | `newsroom`              | `/newsroom` |
| Featured | Data file + custom React page (showcase style) | n/a                     | `/featured` |

**Why this split:**
- All three content sections (Research, Insights, Newsroom) are chronological, tag-able, and benefit from RSS feeds → blog plugin is the right fit. Research posts can still be long-form MDX (blog posts have no length limit); readers get a dated index page per section. Three separate blog instances give each its own index, tag pages, pagination, and feed.
- **No instance uses `/blog`.** The preset's blog slot is remapped to `/insights`; the other two are explicit plugins with their own `routeBasePath`. The path `/blog` stays unused.
- **No docs plugin** is configured — removes the sidebar/versioning machinery that isn't needed here.
- Featured items are third-party external links with rich metadata (logo/screenshot, tags, outlet, date, URL). A TypeScript data file + a custom React page at `/featured` renders a **docusaurus.io/showcase-style** layout: filterable card grid, tag multi-select, search box within the page. No MDX file per item.

### Preset configuration

Use `classic` preset with:
- `docs: false` — no docs plugin on the site.
- `blog: { id: 'default', routeBasePath: '/insights', path: './insights', blogTitle: 'Insights', blogDescription: '…', showReadingTime: true, feedOptions: { type: ['rss','atom'], title: 'Insights' } }` — this is the Insights instance.
- `pages: true` — for the homepage, About, and Featured custom pages.
- `theme: { customCss: './src/css/custom.css' }`.

Then register additional plugins:
- `['@docusaurus/plugin-content-blog', { id: 'research', routeBasePath: '/research', path: './research', blogTitle: 'Research', blogDescription: 'Long-form studies and analyses', showReadingTime: true, blogSidebarTitle: 'All research', blogSidebarCount: 'ALL', feedOptions: { type: ['rss','atom'], title: 'Research' } }]`
- `['@docusaurus/plugin-content-blog', { id: 'newsroom', routeBasePath: '/newsroom', path: './newsroom', blogTitle: 'Newsroom', blogDescription: 'Announcements & milestones', feedOptions: { type: ['rss','atom'], title: 'Newsroom' } }]`
- `['docusaurus-plugin-plausible', { domain: 'your-domain.com' }]` (placeholder until final domain).

> **Note on the Research "blog" URLs:** blog instance post URLs follow `routeBasePath/YYYY/MM/DD/slug` by default. We'll override with `slug:` frontmatter on each post to produce clean `/research/<slug>` URLs, matching the IA above.

## 4. Tech Stack (defaults — confirm in open questions)

| Choice          | Value                                                                                                                                                                                                                      | Notes                                                                                                                                        |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| Docusaurus      | latest stable (v3.x)                                                                                                                                                                                                       | `create-docusaurus` scaffold                                                                                                                 |
| Language        | **TypeScript**                                                                                                                                                                                                             | `docusaurus.config.ts`, `.tsx` components                                                                                                    |
| Package manager | **npm**                                                                                                                                                                                                                    | Default; `package-lock.json`                                                                                                                 |
| Node            | `>=18.0` (Docusaurus requirement)                                                                                                                                                                                          | Specify in `package.json#engines`                                                                                                            |
| MDX             | v3 (Docusaurus default)                                                                                                                                                                                                    | **All content authored as `.mdx`** (Research posts, Insights posts, Newsroom posts, About, homepage fragments where useful). `.md` not used. |
| Styling         | Infima (built-in) + custom CSS variables for brand palette                                                                                                                                                                 | Dark mode enabled                                                                                                                            |
| Search          | **`@easyops-cn/docusaurus-search-local`** — indexes all three blog instances (Research, Insights, Newsroom) and static pages. Zero external service, works offline, free. Algolia DocSearch remains a future upgrade path. |
| Analytics       | **Plausible** via community plugin `docusaurus-plugin-plausible`. Domain placeholder until final domain picked; plugin tolerates empty domain in dev.                                                                      |
| Deployment      | **GitHub Pages** via GitHub Actions workflow (`actions/configure-pages` + `actions/deploy-pages`). `CNAME` file supported in `static/` once custom domain is set.                                                          |
| Comments        | Deferred                                                                                                                                                                                                                   | Giscus is easy to add later on Insights/Research posts                                                                                       |

## 6. Implementation Steps

1. **Scaffold**
   - `npx create-docusaurus@latest content-site classic --typescript` run from repo root (the existing empty `content-site/` needs to be removed or the scaffold run into a temp dir and files moved — scaffolder won't overwrite a non-empty dir). Since only `README.md` is inside, preserve it by moving aside during scaffold, then restore.
2. **Clean defaults**
   - Rename default sample `blog/` directory to `insights/`.
   - Remove sample `docs/` directory and `sidebars.ts`.
   - Remove `src/pages/markdown-page.md`.
   - Remove `src/components/HomepageFeatures` (replaced by our own components).
3. **Configure `docusaurus.config.ts`**
   - Set `title`, `tagline`, `url` (placeholder `https://your-domain.com`), `baseUrl: '/'`, `organizationName` (GitHub user/org placeholder), `projectName: 'abpiv-personal-brand'`, `favicon`.
   - `trailingSlash: false` (GitHub Pages-friendly with explicit URL handling).
   - `onBrokenLinks: 'throw'`, `onBrokenMarkdownLinks: 'throw'`.
   - Preset `classic`: `docs: false`, `blog: { id: 'default', routeBasePath: '/insights', path: './insights', blogTitle: 'Insights', blogDescription: '…', showReadingTime: true, feedOptions: { type: ['rss','atom'], title: 'Insights' } }`.
   - `plugins`:
     - `['@docusaurus/plugin-content-blog', { id: 'research', routeBasePath: '/research', path: './research', blogTitle: 'Research', blogDescription: 'Long-form studies and analyses', showReadingTime: true, blogSidebarTitle: 'All research', blogSidebarCount: 'ALL', feedOptions: { type: ['rss','atom'], title: 'Research' } }]`
     - `['@docusaurus/plugin-content-blog', { id: 'newsroom', routeBasePath: '/newsroom', path: './newsroom', blogTitle: 'Newsroom', blogDescription: 'Announcements & milestones', feedOptions: { type: ['rss','atom'], title: 'Newsroom' } }]`
     - `['docusaurus-plugin-plausible', { domain: 'your-domain.com' }]` (placeholder — swap when domain known; plugin is a no-op in dev).
   - `themes`: `[['@easyops-cn/docusaurus-search-local', { hashed: true, indexDocs: false, indexBlog: true, indexPages: true, blogRouteBasePath: ['/research', '/insights', '/newsroom'], highlightSearchTermsOnTargetPage: true }]]`.
   - `themeConfig`:
     - `navbar.items` (in order): Research (`to: '/research'`), Featured (`to: '/featured'`), Insights (`to: '/insights'`), Newsroom (`to: '/newsroom'`), About (`to: '/about'`), GitHub icon on right.
     - `footer`: socials + RSS links (`/research/rss.xml`, `/insights/rss.xml`, `/newsroom/rss.xml`).
     - `colorMode`: `respectPrefersColorScheme: true`.
     - `metadata` + OG image (`static/img/social-card.png`).
4. **Research blog instance**
   - Create `research/` directory with `authors.yml`, `tags.yml`, one seed post `2026-04-30-intro.mdx` with frontmatter: `slug: intro`, `title`, `authors`, `tags`, `description`. Seed post demonstrates longform MDX (headings, callouts, code blocks).
   - Using `slug: <clean-slug>` in every post frontmatter forces URLs to `/research/<slug>` instead of the default date-based path.
5. **Insights + Newsroom seeds**
   - One `.mdx` post each with frontmatter (`slug`, `title`, `authors`, `tags`, `date`, `description`).
   - `authors.yml` with one author entry (the user) — replicated into all three blog instance folders.
6. **Featured (docusaurus.io/showcase-style)**
   - `src/data/featuredTags.ts`: exports a `TagType` union + `Tags` record mapping each tag key to `{ label, description, color }`. Mirrors the pattern in `docusaurus/website/src/data/tags.tsx`.
     - Proposed tag set: `podcast`, `interview`, `article`, `guest-post`, `talk`, `keynote`, `video`, `panel`, `newsletter`, `book`, `favorite`.
   - `src/data/featured.ts`: exports `FeaturedItem[]` where each item is `{ title, outlet, description, preview: require('@site/static/img/featured/...'), website: string, tags: TagType[], date: 'YYYY-MM-DD' }`. `preview` is a screenshot or outlet logo.
   - `src/pages/featured/index.tsx`:
     - Page-level hero: title "Featured", short intro paragraph, primary CTA ("Suggest an appearance" → mailto or GitHub issue).
     - `FeaturedFilters` bar: tag multi-select (OR/AND toggle), text search input, operator toggle, clear button. State reflected in URL query string (`?tags=podcast,talk&operator=AND&q=retrieval`) so filters are shareable — same UX as Docusaurus showcase.
     - `FeaturedCard`: square/16:9 preview image, title, outlet, tags, external-link icon. Whole card is an anchor to `website` with `target="_blank" rel="noreferrer"`.
     - Sorting: "favorite" tagged items pinned to top, then newest-first by `date`.
     - Grid: responsive CSS grid, 1/2/3/4 columns depending on breakpoint.
     - Uses `@docusaurus/Head` to set page title + OG metadata.
     - No Algolia dependency — in-page filtering is pure client-side React over the data array.
7. **Homepage**
   - `src/pages/index.tsx`: `HomeHero` (name, tagline, primary CTAs → Insights + About), `SectionCards` (four cards linking to Research / Featured / Insights / Newsroom with short blurbs), `LatestPosts` component rendering the 3 newest posts from each of Research/Insights/Newsroom (sourced from `@docusaurus/plugin-content-blog`'s `globalData` via `useGlobalData()` / `usePluginData('docusaurus-plugin-content-blog', '<id>')`).
8. **Search**
   - Install `@easyops-cn/docusaurus-search-local` (added in step 3's `themes` block).
   - Config indexes all three blog instances via `blogRouteBasePath`; docs indexing disabled.
   - Smoke-test: navbar search returns hits from each of Research, Insights, Newsroom, and static pages (About, Featured).
9. **Analytics**
   - `docusaurus-plugin-plausible` registered in step 3. Domain field is a placeholder (`your-domain.com`) until the real domain is known — plugin doesn't emit real events until the deployed site matches the configured domain, so dev/preview builds are safe.
10. **Branding / CSS**
    - Update `src/css/custom.css`: brand primary color scale (placeholder palette — will replace when brand colors arrive), font stack (system fonts by default, swap to custom webfont later), heading weight, container max-width tweaks. Dark-mode variants defined in the same file under `[data-theme='dark']`.
    - Replace `static/img/logo.svg` and `favicon.ico` with provided assets or keep placeholders.
11. **Scripts & tooling**
    - `package.json` scripts: `start`, `build`, `serve`, `clear`, `typecheck`, `write-heading-ids`.
    - `.gitignore`: `.docusaurus/`, `build/`, `node_modules/`.
    - `.nvmrc` or `engines.node` pinned to an LTS satisfying Docusaurus requirements.
12. **Build verification**
    - `npm install && npm run build` must pass with zero broken links and `npm run typecheck` clean before the work is considered done.
    - Manually smoke-test: navbar search returns results from all three blog instances; Featured page filters/search work; every section link in the navbar renders; RSS feeds at `/research/rss.xml`, `/insights/rss.xml`, `/newsroom/rss.xml` are valid XML.
13. **Deployment — GitHub Pages**
    - `.github/workflows/deploy.yml`:
      - Trigger: push to `main` touching `content-site/**` + manual `workflow_dispatch`.
      - Job 1 (build): Node setup, `npm ci` in `content-site/`, `npm run build`, upload `content-site/build` via `actions/upload-pages-artifact`.
      - Job 2 (deploy): `actions/deploy-pages` with `github-pages` environment.
      - Permissions: `contents: read`, `pages: write`, `id-token: write`.
    - `static/CNAME` added when custom domain is confirmed (content = the apex or subdomain).
    - If deploying to `<user>.github.io/abpiv-personal-brand/`, `baseUrl` must be `'/abpiv-personal-brand/'`; if apex custom domain, `baseUrl` stays `'/'`. Plan assumes apex/custom domain with `baseUrl: '/'` — **verify before first deploy**.
